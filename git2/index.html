VERSION CONTROL

Version control systems are a category of software tools that help a software team manage changes to source code over time. Version control software keeps track of every modification to the code in a special kind of database. If a mistake is made, developers can turn back the clock and compare earlier versions of the code to help fix the mistake while minimizing disruption to all team members.

THREE COMMANDS USED IN GIT

  1. Git clone

    2. Git clone is a command for downloading existing source code from a remote repository (like Github, for example). In other words, Git clone basically makes an identical copy of the latest version of a project in a repository and saves it to your computer.

  2. Git commit

    3. This is maybe the most-used command of Git. Once we reach a certain point in development, we want to save our changes (maybe after a specific task or issue).Git commit is like setting a checkpoint in the development process which you can go back to later if needed.

  3. Git push

    4. After committing your changes, the next thing you want to do is send your changes to the remote server. Git push uploads your commits to the remote repository.Git push only uploads changes that are committed.
